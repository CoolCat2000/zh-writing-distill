# 中文写作风格蒸馏方法

本文档用于从 `docs/` 目录中的中文文章提炼个人写作风格，并生成可执行的 `AGENTS.md` 写作规则。

## 分析原则

- 从素材归纳，不从模板套结论。
- 先找反复出现的稳定特征，再写规则。
- 区分“内容主题”和“表达方式”：不要把作者常写 AI、产品、生活，误判成文风。
- 对高影响结论保留依据：引用短句、标题、文件名或多个文件的共同现象。
- 生成结果要能指导下一次写作，而不是只描述过去文章。

## 素材盘点

读取 `docs/` 后，先建立素材概览：

- 文件数量、文件类型、标题或首行。
- 主题分布：常写什么，不常写什么。
- 文体分布：长文、随笔、教程、评论、复盘、故事、说明文等。
- 第一人称使用情况：是否常用“我”“我们”“你”。
- 文章长度和段落密度：短段为主、长段为主，还是混合。

如果素材少于 3 篇，生成规则时标注“样本有限”，避免过度泛化。

## 蒸馏维度

### 写作身份

提炼作者在文章中的默认位置：

- 是亲历者、观察者、解释者、讲故事的人，还是方法论提供者。
- 是平视交流、专业讲解、犀利评论，还是温和陪伴。
- 是否常把个人经验放在公共议题前面。

输出时写成后续 agent 能执行的身份规则，例如：

```markdown
写作时站在“亲自试过的人”的位置说话。先交代自己的观察和感受，再扩展到方法或判断。
```

### 核心价值取向

从反复出现的判断、好恶和取舍中提炼 3 到 5 条价值取向：

- 作者赞成什么。
- 作者警惕什么。
- 作者反对什么。
- 作者认为读者真正需要什么。

每条用一句规则表达，并附素材依据。

### 语气与姿态

观察作者如何和读者说话：

- 直接还是委婉。
- 克制还是情绪外露。
- 是否自嘲、吐槽、反问。
- 是否承认不确定性。
- 是否喜欢用口语打断严肃论述。

生成规则要具体到可模仿动作：

- 可以使用哪些口语表达。
- 什么时候可以短句独立成段。
- 什么时候必须收住情绪，回到事实。

### 句式与节奏

统计并归纳：

- 长句和短句的比例。
- 是否大量一句话成段。
- 是否使用连续追问。
- 是否用逗号制造口语停顿。
- 是否偏好并列、递进、转折或回环。

不要只写“节奏自然”。要写清楚如何做到自然。

### 结构习惯

观察文章常见骨架：

- 从故事进入，还是从结论进入。
- 是否先提出困惑，再展开探索。
- 是否经常“现象 -> 解释 -> 反转 -> 判断”。
- 结尾是行动建议、情绪回扣、开放问题，还是明确判断。

如果存在多种文章原型，可以分成 2 到 5 类，每类写适用场景和结构。

### 词汇偏好

提炼：

- 高频词、口头禅、标志性短语。
- 常用动词、形容词和连接方式。
- 偏好的中文标点。
- 喜欢或避免的专业术语。

词汇规则要分成“推荐使用”和“谨慎使用/禁止使用”。

### 证据处理

`AGENTS.md` 的“素材依据”部分保留简短证据：

- 每条证据尽量不超过 30 个汉字。
- 标明来源文件名。
- 可以用“多篇文章反复出现”描述跨文件现象。
- 不要长篇复制原文。

## AGENTS.md 写法

生成的是规则文件，不是研究报告。优先使用这种句式：

- “写作时……”
- “优先……”
- “避免……”
- “当素材不足时……”
- “交付前检查……”

避免这种空泛句式：

- “作者风格鲜明。”
- “语言具有感染力。”
- “整体读起来很自然。”
- “兼具理性与感性。”

## 自检清单

最后生成 5 到 10 条自检规则。每条必须能直接判断：

- 是否像作者。
- 是否过度解释。
- 是否缺少个人视角。
- 是否用了素材中少见的腔调。
- 是否把观点写成了通用鸡汤。

示例：

```markdown
- 是否先给出真实观察，再给出抽象判断。
- 是否有至少一个来自个人经验或具体场景的细节。
- 是否删掉了“重要意义”“持续赋能”“深度洞察”这类泛化表达。
```
